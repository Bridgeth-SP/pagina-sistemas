Modelo OSI

Capas del Modelo OSI de la más alta a la más baja:

Open Systems Interconnection

El modelo OSI, siglas de "Open Systems Interconnection" (Interconexión de Sistemas Abiertos), es un modelo conceptual que proporciona una estructura para entender y describir cómo funcionan las redes de computadoras. Fue desarrollado por la Organización Internacional de Normalización (ISO) en la década de 1980.

Capa 7

Es la capa más alta del modelo y proporciona servicios directamente a las aplicaciones. Incluye protocolos específicos para aplicaciones como el correo electrónico (SMTP), transferencia de archivos (FTP) y navegación web (HTTP).

Capa 6

Capa 6: Presentación

Capa 5

Capa 5: Sesión

Capa 4

Proporciona servicios de transporte de extremo a extremo, asegurando que los datos se entreguen de manera fiable y en el orden correcto. Esta capa también se encarga de dividir y reensamblar los datos en segmentos más pequeños, si es necesario.
Capa 4: Transporte

Capa 3

Gestiona el enrutamiento de datos a través de la red. Se encarga de determinar la ruta más eficiente para enviar los datos desde el origen al destino.
Capa 3: Red

Capa 2

Proporciona un enlace fiable y libre de errores entre dos nodos de red adyacentes. Se encarga de la detección y corrección de errores, así como del control de flujo.
Capa 2: Enlace de datos

Es la capa más baja del modelo y se encarga de la transmisión física de los datos a través del medio de comunicación, como cables o señales inalámbricas.

Modelo de red TCP/IP

Protocolo de Control de Transmisión/Protocolo de Internet (Transmission Control Protocol/Internet Protocol)

Es un conjunto de protocolos de red ampliamente utilizado que permite la comunicación y el intercambio de datos en Internet. Fue desarrollado en los años 70 y se convirtió en el protocolo estándar para la comunicación en redes de área amplia (WAN) y redes de área local (LAN).
Capa 1
Acceso al
medio

Capa 1

También conocida como Capa de Interfaz de Red, esta capa se encarga de la interconexión física de los dispositivos de red con el medio de transmisión, como cables o conexiones inalámbricas. Define las especificaciones técnicas y los estándares para la transmisión de datos en el nivel físico y de enlace de datos.
Capa 2
Internet

Capa 2

Esta capa se ocupa del enrutamiento y direccionamiento de los paquetes de datos a través de la red. Se encarga de establecer la conexión entre diferentes redes y garantizar que los datos lleguen al destino correcto. El protocolo principal utilizado en esta capa es el Protocolo de Internet (IP).
Capa 3
Transporte

Capa 3

Esta capa se encarga de garantizar la entrega fiable de los datos y el control del flujo de información entre los dispositivos finales. El protocolo principal utilizado en esta capa es el Protocolo de Control de Transmisión (TCP), que divide los datos en segmentos, los envía y los reensambla en el destino para asegurar una entrega sin errores.
Capa 4
Aplicación

Capa 4

Esta capa es la más cercana a las aplicaciones de usuario y proporciona los servicios de comunicación para aplicaciones específicas. Aquí se encuentran los protocolos como HTTP (Hypertext Transfer Protocol) para la navegación web, FTP (File Transfer Protocol) para la transferencia de archivos, SMTP (Simple Mail Transfer Protocol) para el envío de correos electrónicos, entre otros.
